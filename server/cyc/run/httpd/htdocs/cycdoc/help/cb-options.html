This page allows the user to set interface options for the KB Browser.

The Buttons

[Cancel] Cancel all options changes and return to the previous page.
[Reset Form] Restore all options settings to their values before changes were made.
[Submit] Set options for the current session only, reflecting any changes to the various options made on the Browser Options page.
[Submit & Save] Like [Submit], but saves settings for use in future sessions. Saved settings get loaded whenever you log in to an image (provided they can be found), or when you click on...
[Load Saved] Load your most recently saved options.

General Options

  • Wrap Assertions: clicking on this option causes assertions to wrap when displayed by the KB Browser. The default value is for no wrap to occur, which allows more assertions to be displayed at one time, but may inhibit viewing of an entire axiom due to length.
  • Wrap Interactor and Ask input boxes: clicking on this options causes text in the large text input windows (e.g Interactor, Ask, Assert, Edit) to wrap automatically. The default value is for no wrap to occur.

  • Show documentation in separate window:
  • Show assertion ball tooltips:
  • Show alternating shades of grey backgrounds in tables:
  • Try grepping if other Show methods fail: when this is set, the system will try grepping on a string entered into the completion box if all other methods of locating an appropriate term fail.
  • Status Frame Update: setting this option to an integer value causes the Agenda Status Bar to be automatically updated in intervals of the specified number of seconds. It may be necessary to refresh the KB Browser or click on the "Update" link in the Agenda Status Bar one final time to activate this option. Leaving the input field blank (the default value) disables the automatic update; the Agenda Status Bar must then be updated manually be clicking on the "Update" link when an update is desired.
  • Frames and JavaScript: to use HTML frames and JavaScript while using the KB Browser, make sure both buttons are clicked. Both are recommended and are set as default values. Unfortunately, some HTML browsers do not yet support HTML frames and/or JavaScript. If this is the case with your browser, unclick the relevant option to see a more basic version of the KB Browser.

Term Display Options

The term display options affect how a CYC® term is displayed in the Browser.

  • Default Content: this option sets what kind of information is displayed for a given term when that term is first selected. It is only possibly to choose one of the listed options at a time.
    • As much as possible up to __ assertions: will display as many assertions as possible up to the number specified, regardless of type of assertion.
    • Documentation: will display documentation assertions about the term, including #$comment, etc.
    • Definitional Information: will display assertions considered definitional for the term, such as #$isa, #$genls, etc; will also display whether a term is "public" or "mentioned in code".
    • Lexical Information: will display lexical information about the term, including #$denotation, etc.
  • Assertion Formulas: this option sets how assertions are displayed when viewing a browser page for a Cyc term. It is only possibly to choose one of the listed options at a time.
    • HL formulas: will display the HL formula of each assertion, including such things as #$termOfUnit and skolem terms.
    • EL formulas: will display the EL formula of each assertion, which is computed at display time, making this option somewhat slower.
    • EL formulas except when browsing skolems: will display EL formulas, except when you are browsing skolem terms, which generally are not included in EL formulas.
  • Assertion Display Options: These control how assertions are displayed when viewing them separately, in an Assertion Browser page. More than one option can be chosen.
    • Show EL formula: will display the EL formula of the assertion, which is computed at display time, making this option somewhat slower.
    • Show HL formula: will display the HL formula of the assertion, including such things as #$termOfUnit and skolem terms.
    • Show English translation: will display a natural-language (typically English) paraphrase of the assertion.
    • Always perform access diagnostics: will run diagnostic tests on each assertion you view.

History and Browser Tool Options

  • History Display Options: These options control what is displayed on the History page.
    • Display no more than __ constants: sets a limit on how many recently browsed constants to display. As you view new ones in excess of the limit, the oldest get bumped off the history list.
    • Display no more than __ NARTS: ditto, for non-atomic reified terms.
    • Display no more than __ assertions: ditto, for assertions.
    • Show EL assertion formulas: whether or not to compute and display EL formulas for assertions.

NL Options

  • NL Generation:
    • Show assertions in English instead of CycL: causes the NL paraphrases of assertions (typically in English) to be displayed instead of their EL or HL formulas.
    • Assume Cyc is addressing the current cyclist: causes the paraphrase code to do things like using "me" and "you" to refer to Cyc and the currently logged-in Cyclist.
  • Lexification:
    • Reviewer: the Cyclist who is to review your lexical assertions. Leave blank if your lexical assertions do not require review.
    • Default Microtheory: where your lexical assertions are placed.
    • Default Paraphrase Microtheory: where your paraphrase assertions are placed.
    • Default Parse Template Microtheory: where your parse template assertions are placed.
