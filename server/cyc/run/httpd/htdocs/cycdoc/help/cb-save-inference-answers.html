Available Formats

  • Text
    The results shall be saved such that all the bindings of a particular result are tab-delimited and each result is on a separate line.

  • Default XML
    The results shall be saved as XML using the default XML specification. As a result, there shall be a separate <binding> element for each of the query results. The results are stored within each of these <binding> elements with the EL-variable being the child element and its binding being the text-value of the child element.

  • User-defined XML
    The results shall be saved as XML using the XML specification that the user provides.

Steps to Save Results

  1. Select the format in which the results must be stored and click on the [Submit] button.
    Note: If you want the results to be stored as User-defined XML, then please make sure that a valid XML specification is provided.
  2. The results are now displayed.
    Note: Depending upon the way your browser is set up, the results may or may not be displayed in the required format. It is very likely that the results are NOT displayed in the required format, but are bunched together. This is okay.
  3. Click on the menu File and then Save Page As ...(or Save As ...). This shall bring up a dialog box.
  4. In the dialog box,
    • Mozilla users: Please make sure that All Files (*) is selected as the file type.
    • Internet Explorer users: To save as text, please make sure that Web Page, HTML only (*.htm, *.html) is selected as the file type. To save as XML, please make sure that XML files (*.xml) is selected as the file type.
    • Firefox users: Please make sure that Web Page, HTML only (*.htm, *.html) is selected as the file type.
    Select a location for the file and enter a file-name. Then click on the [Save] button.
  5. Finally, close the window.

User-defined XML Specification

The XML specification that is used to convert the query results to XML, is of the following format: 
(:name element-name 
 [:sort-asc|:sort-desc sort-key] 
 [:attributes (att-name1 att-val1 att-name2 att-val2 ...)]
 [:text element-val]
 [:elements (child1 child2 ...)])
where
    element-name must be a valid string
    element-val must be a valid string or an EL-variable used in the query
    sort-key must be an EL-variable used in the query
    att-name must be a valid string
    att-val must be a valid string or an EL-variable used in the query
    child must be a valid XML element specification in the above format

Please note that the keyword-value pairs within square brackets are optional. These optional pairs must be separated by a whitespace. The order of these optional pairs is irrelevant.

Example: For the following query

(#$thereExists ?ALLOT
  (#$and
    (#$equals ?PROJECTMANAGER #$Klein)
    (#$projectManagers ?PROJECT ?PROJECTMANAGER)
    (#$projectAssignments
     (#$SpecificationOfProjectEventFn ?PROJECT) ?ASSIGNMENT)
    (#$hasAllotment ?ASSIGNMENT ?ALLOT)))
here's a valid XML specification: 
(:name "project" :sort-asc ?PROJECT
 :attributes ("id" ?PROJECT "manager" ?PROJECTMANAGER)
 :elements
 ((:name "assignment" :sort-desc ?ASSIGNMENT
   :text ?ASSIGNMENT)))
