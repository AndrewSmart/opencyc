The SubL Interactor is an input window for evaluating SubL expressions.

Evaluating Expressions

To evaluate an expression, simply enter the expression in the window and press the [Eval] button. This will bring up a new frame which contains the last form (the expression just evaluated), the results of the evaluation, and a fresh input window. If the expression is malformed an error message is returned.

To get constant completion while typing in an expression, press the [Complete] button in the middle of a constant name. As always, possible completions (if there is more than one) will be displayed in a separate completion window which will be reused for future completions. Completion is case sensitive, and requires that JavaScript is enabled in your browser.

To completely refresh the input window and start over, press the [Clear] button.

To return to the previous frame, press the [Back] button.

Function Documentation

To view the documentation and arglist for any SubL function, click on "Show function documentation", type in an incomplete function expression, and click on the [Eval] button. Instead of trying to evaluate the expression, the Interactor will find the first incomplete function call it can and return the arglist along with any explicit documentation. For example,

        (ke-assert
entered into the input window (with "show function documentation" on) will return: 
        KE-ASSERT : (FORMULA MT &OPTIONAL STRENGTH DIRECTION)
        [documentation would go here if there were any]
Note that this feature is not available in all versions of the CYC® System.

At certain times (such as when function documentation is shown), the form originally entered is returned in the input window; if new text is then entered into the window, pressing the [Reset] button will replace the newly entered text with the original returned text. This option is only available if text has been returned in the window by the Interactor.

For more information about SubL, read the SubL Reference Documentation.
